# Monitoring Operations

## Quick Start
To start the monitoring stack locally, run the following from the `nem.infrastructure/` directory:

```bash
docker compose up -d
```

This starts all LGTM components (Loki, Grafana, Tempo, Prometheus) along with the exporters included in this stack.

## Accessing Grafana
Grafana is the central portal for all monitoring data.

*   **URL**: [http://localhost:3010](http://localhost:3010)
*   **Credentials**: `admin` / `admin` (Default)
*   **Default Dashboards**: Provisioned on startup.

## Dashboard Guide
The following core dashboards are available in the `Dashboards` menu:

1.  **Infrastructure Overview**: Displays system health, container resource usage (CPU/Memory), and database/container visibility. Metrics are sourced from `cadvisor` and `postgres-exporter`.
2.  **RED Metrics**: Shows Rate, Error, and Duration for all Instrumented services. Includes quantile distributions (P50, P90, P99) and error rate trends.
3.  **Log Explorer**: A specialized view for searching and filtering logs from Loki. Filter by `service_name`, `level`, or search for specific message content using LogQL.
4.  **Trace Explorer**: Provides trace searching and visualization. Allows finding slow requests, viewing call hierarchies, and inspecting service maps generated by Tempo.

## Alert Management
Alerts are managed via Grafana Alerting and provisioned through YAML files.

### Key Alert Rules
*   **service-down**: Fires when `up == 0` for more than 2 minutes. Critical severity.
*   **error-rate-spike**: Fires when the 5xx HTTP error rate exceeds 5% for 5 minutes. Warning severity.

### Alert Routing
Alerts are routed to the `nem.Comms` webhook, which dispatches to configured supported channels such as Telegram, Teams, WhatsApp, Signal, or WebWidget.

## Troubleshooting
*   **Service Missing in Prometheus**: Check if the service is running and `Enabled: true` is set in its `appsettings.json`. Verify the `/metrics` endpoint is reachable.
*   **No Logs in Loki**: Ensure the service has OTLP logging enabled and the OTEL collector is reachable on port 4317. Check collector logs for export errors.
*   **No Traces in Tempo**: Verify `ActivitySource` names match the configured service name. Traces are batched; it may take 5-10 seconds for them to appear.
*   **Grafana Dashboards Empty**: Ensure the data sources (Prometheus, Loki, Tempo) are healthy in the `Connections > Data Sources` menu.

## Stack Management
Commands for managing the monitoring infrastructure:

```bash
# Start the stack
docker compose up -d

# Stop and remove containers
docker compose down

# View logs for a specific component
docker compose logs -f otel-collector

# Restart a component after config change
docker compose restart prometheus
```

## Additional Context

### Integration with nem.* Ecosystem

This component integrates with the broader nem.* microservice ecosystem through standard platform conventions:

- **Messaging**: Wolverine handlers for asynchronous command and event processing via RabbitMQ
- **Authentication**: Keycloak-issued JWT tokens validated on all API endpoints
- **Authorization**: OPA policy evaluation with fail-closed default behavior
- **Observability**: OpenTelemetry instrumentation for distributed traces, metrics, and structured logging
- **Configuration**: `IConfigurationManager` pattern for runtime-configurable settings via nem.MCP
